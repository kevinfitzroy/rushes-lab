# 权限模型(产品视角)

> 这份文档解释你"为什么能/不能看到这个东西",不涉及底层技术(OpenFGA / SQL)。
> 想深入技术细节去仓库 `rushes-spec/material-storage/permissions-model-v4.md`。

---

## 0. 一句话总览

**项目可见性** + **项目内角色** + **敏感文件夹邀请** + **临时授权** 四层叠加决定你能做什么。

---

## 1. 项目可见性(谁能"看到这个项目存在")

| 可见性 | 谁能在项目列表看到 metadata | 谁能进去看内容 |
|---|---|---|
| **public** | 所有登录用户 | 任意登录用户(等同 viewer);可申请更高权限 |
| **private** | 仅项目成员 | 仅项目成员 |

`demo-onboarding` 演示项目就是 public,所以所有人都能进。

---

## 2. 项目内角色(成员的 4 个 level)

| 角色 | 能干什么 | 通常给谁 |
|---|---|---|
| **viewer** 浏览者 | 看 metadata + 文件名,**不能下载** | 临时观察、品牌方 |
| **downloader** 下载者 | 看 + 下载 | 客户、合作方 |
| **uploader** 上传者 | 看 + 下载 + 上传 + 新建文件夹 | 拍摄/剪辑团队 |
| **admin** 管理员 | 以上全部 + 改成员 + 邀请别人进敏感文件夹 + 改项目设置 | 项目负责人 |

**角色继承规则**:

- 管理员包含上传者权限,上传者包含下载者,下载者包含浏览者(向下兼容)
- 一个人**可以同时被赋予多个角色**(如管理员 + 下载者),按最高权限算
- 角色可以赋予**个人**、**用户组**、**部门**(部门成员会自动继承,包括子部门)

**普通文件夹(不带 🔒)**:权限**完全继承自所在项目**。如果你是项目 downloader,你就能下载这个项目里所有普通文件夹下的任何文件。

---

## 3. 敏感文件夹(🔒)— 完全独立的权限

敏感文件夹设计用来管"客户原片/术前术后照/合规要求隔离的内容"。它跟普通文件夹有 3 个本质区别:

### 3.1 不继承项目角色

即使你是这个项目的**管理员**,你也**看不到**任何敏感文件夹的内容,除非你被**显式邀请**进去。

> 例外:你自己新建的敏感文件夹会自动给你 viewer 权限 — 否则会"自己建完自己看不到"。这是 2026-05-17 修的(issue #87)。

### 3.2 邀请等级只有 2 个

| 邀请等级 | 能干什么 |
|---|---|
| **viewer** | 看缩略图 + 文件名 |
| **downloader** | 看 + 下载 + 上传(敏感文件夹的上传 = downloader 等级) |

### 3.3 文件夹管理权(can_admin)仍跟项目走

谁是项目 admin,谁就是这个项目下所有敏感文件夹的 admin(能邀请别人 / 撤销邀请)。但 admin 自己若要**看**内容,需要单独给自己 invite。

### 3.4 结构限制

- 敏感文件夹**只能直接挂在项目下**,不能嵌在普通文件夹里
- 敏感文件夹**下面不能再建子文件夹**(限一级)
- 这是底层 model 强制的,不是 UI 限制

---

## 4. 临时授权(审批后的限时权限)

正常的角色/邀请是**永久**的。临时授权是审批通过后给的**有时效**权限,常见场景:

| 场景 | 申请的是 | 时长建议 |
|---|---|---|
| 客户要看一个具体文件 | asset 单文件下载权 | 24h |
| 客户要看一段敏感文件夹的内容 | sensitive_folder access(viewer/downloader) | 24h - 7d |
| 加入一个新项目 | project viewer/downloader | 永久(=升级为正式成员) |

申请流程:

1. 你看到没权限的资源 → 弹 "申请访问" 框 → 填理由 + 时长
2. 后端写一条 `ApprovalRequest`,推飞书 IM 卡片给项目 admin
3. admin 在飞书卡片上点"批准" → 后端写一条临时 grant(带过期时间)
4. 你立刻有权限了;在 "我的权限" → "临时授权" 段能看到剩余时长

到期后授权自动失效。

---

## 5. 单文件分享(短链)

跟"申请权限"不一样,短链是**你主动给已经有下载权的人传一个文件**:

- 在文件右栏点"分享" → 生成 `/s/<token>` 短链
- 默认要求对方**登录素材库**才能打开(防止 token 泄漏被外人捡)
- 同时推一张飞书卡片给被分享人,卡片里有"打开"按钮
- 短链有效期默认 24h,可改

短链**不是**临时授权 — 短链 user 必须已经有 can_view/can_download(通过角色或临时授权)才能真正下载。

---

## 6. 系统管理员(很少人)

系统管理员 = `organization#admin`,头像旁有 `SYSTEM ADMIN` 金标。能力:

- 创建/删除任意项目
- 看全局 audit 日志(`/admin/audit`)
- 任何资源直通(包括敏感文件夹,不需要邀请)
- 审批别人不能审批的申请(2026-05-17 修补)

系统管理员**不会被普通 UI 操作授予** — 只能 dev 在后台 OpenFGA 直接 grant。所以普通用户不用考虑成为系统管理员。

---

## 7. 一张"我能不能"决策树

想做某件事但不确定有没有权限?按这个走:

```
我想 [操作] 这个 [资源]
  ↓
1. 这个 [资源] 我看得到吗?
   - 看得到 → 继续 2
   - 看不到 → 项目 visibility 私有 + 我不是成员;或者它在敏感文件夹我没被邀请
   ↓
2. 这个操作需要的权限等级我有吗?
   - 看 metadata → viewer 够
   - 下载文件 → 需要 downloader / admin / 临时授权 / explicit_downloader
   - 上传新文件 → 需要 uploader / admin
   - 改成员 / 改设置 / 邀请别人进敏感文件夹 → 需要 admin
   ↓
3. 没有 → 走"申请访问"流程(§4),写明理由让项目 admin 决策
```

---

## 8. 我已经有权限了为什么还说"权限不足"?

几个常见 false-positive:

- **session 过期**:重新登录就好,不是真的权限不足
- **OpenFGA 同步延迟**:刚被加进成员,等几秒刷新页面
- **看的是敏感文件夹但没被邀请**:走 §4 申请
- **错误文案误导**:有时后端文案不够精确(如 GroupPicker "无匹配项" 历史上写过"权限不足");遇到反馈给 gatekeeper
