# 关于这个敏感文件夹

你现在看到这份文档,说明你**已经有这个敏感文件夹的访问权**(viewer 或 downloader)。

## 这里是什么

这是 **demo-onboarding** 演示项目下的一个 **敏感文件夹**(🔒)。

它的存在是为了让你**体验**敏感文件夹的产品行为,不存放真实业务数据。

## 跟普通文件夹的区别

回顾一下(详见 `权限模型.md` §3):

1. **不继承项目角色** — 这个项目你即使是 viewer/downloader/uploader,也看不到这里;必须被项目 admin 单独邀请
2. **结构限制** — 这个文件夹下面不能再建子文件夹
3. **邀请等级只有 2 个** — viewer 或 downloader

## 你可以试什么

### 1. 试一下"看不见敏感文件夹"

退出登录,用一个不是这个项目 admin、也没被邀请进这个 sensitive folder 的账号登录,你应该**看不到**这个文件夹(项目列表里的 demo-onboarding 你看得到,但点进去左栏文件夹树里不会出现"03-敏感示例")。

> ⚠️ 如果你**有系统管理员权限**(头像旁有 SYSTEM ADMIN 金标),敏感文件夹对你直通,以上行为不适用。

### 2. 试一下"申请访问"

未邀请的账号点 demo-onboarding 项目,理论上会有个入口让你申请加入。具体 UI 看 `操作手册.md` §6。

### 3. 试一下"邀请别人"

如果你是这个演示项目的 admin,可以试着在这个敏感文件夹上点"邀请",把同事按 viewer/downloader 加进来,看看对方的视角变化。

---

*这是一个 seed 自动生成的 demo 文件夹,每次部署重建时都会重新生成。不要在里面放真实业务数据 — 数据库重建时它会被清空重新种。*
